## VERIFIED FINDINGS — Fix required

### 1. Experiment 1 does not test Assumption A3 — but our data DOES answer it
**Severity: CRITICAL. Reframing strongly recommended.**

Executive Summary A3 claims: *"~1,500 nuclear imports not all required short-term. Minimal set sustains ATP synthesis without structural/repair proteins."* Exp 1 decays all 769 nuclear genes uniformly, which tests the null case, not A3.

**Verified in data:** Exp 1b gene_knockout_scores.csv already contains the A3 answer:
- **348 / 769 nuclear genes (45%) have zero knockout impact** → dispensable for steady-state ATP
- **91 genes have >90% knockout impact** → essential
- Distinct impact clusters (97.92%: 56 genes; 92.50%: 33 genes; 50.60%: 29 genes; 30.72%: 92 genes) reveal AND-rule multi-subunit complexes where all members are equivalent

**Action:** Reframe Exp 1b's Stage 1 as the headline A3 result. New primary finding: *"A minimal essential set of N nuclear-encoded genes sustains steady-state ATP; ~45% of the 769-gene nuclear import complement is dispensable for short-term function."* This is defensible, mechanistic, and directly answers the project's central hypothesis.

---

### 2. Objective function `OF_ATP_mitoMap` is wrong for transit conditions
**Severity: HIGH. Requires model re-parameterization.**

Verified: `OF_ATP_mitoMap` maximizes cytoplasmic ATP hydrolysis. Mechanism: `ATPtmB_mitoMap` (`0.82 PMF_c + adp_c + atp_m ↔ 0.82 PMF_m + adp_m + atp_c`) exports matrix ATP to cytoplasm, consuming PMF.

For an extracted mitochondrion with NO cytoplasm, this objective is biologically incorrect. The viability criterion is ΔΨm maintenance (PINK1/Parkin threshold, JC-1 staining measures this, not ATP export). MitoMAMMAL already has a `PMF_m` metabolite — it IS the ΔΨm proxy.

**Action:** Add a secondary objective: maximize PMF_m accumulation while blocking `ATPtmB_mitoMap` (no cytoplasmic export). Report both metrics. This also makes Exp 3 (JC-1 validation) directly possible.

---

### 3. FBA reroutes around Complex I knockout — biologically implausible
**Severity: HIGH. Undermines Exp 1b "single-complex" finding.**

Verified manually: setting CI_mitoMap bounds to 0 yields ATP = 69.9 (69% of baseline) via alternate routes. Biologically, Complex I deficiency (Leigh syndrome) is catastrophic — patients die. FBA's steady-state mass-balance allows flux rerouting (CII → CIII → CIV + substrate-level phosphorylation) unavailable in intact mitochondria.

**Consequence:** The Exp 1b "immortalizing any single complex → Δtransit = 0h" finding is an FBA topology artifact, NOT a biological statement. Do not publish as "co-stabilization required" — that's overclaiming.

**Action:** Either add integer/logical constraints forcing sequential flow I→III→IV, or drop the complex-level single-stabilization claim from the abstract. Keep the gene-level essentiality ranking (which is real).

---

### 4. GPR handling bug — latent under uniform decay, manifest under non-uniform
**Severity: MEDIUM now, CRITICAL once empirical half-lives arrive.**

Verified by inspection: `apply_decay_to_model` iterates `for gene in nuclear_genes: for rxn in gene.reactions: rxn.upper_bound = ...`. For a multi-gene AND-rule reaction (e.g., CI_mitoMap has 91 AND-linked genes), the UB gets set 91 times, and the LAST iteration wins.

Under uniform t½ = 12h, all 91 genes compute the same decay factor → all set the same UB → bug has no effect on current results. Under non-uniform half-lives (three-regime or empirical), the bug becomes real: AND rules should take MIN across gene decay factors; OR rules should take SUM.

**Action:** Switch to MitoMAMMAL's shipped `efflux_method.py` `recursive_evaluation()` — it correctly parses GPR with min/sum. Our flux-relative scaling can stay; just replace the gene-loop with proper GPR evaluation.

---

### 5. Flux-relative scaling uses `abs()` on baseline flux — loses reversibility
**Severity: MEDIUM. Affects reversible transporters in Scenario B/C.**

Verified at `experiment1_transit_window.py:162` and elsewhere: `baseline_fluxes = {r.id: abs(sol.fluxes.get(r.id, 0.0)) for r in model.reactions}`.

For a reversible reaction running at baseline flux = -0.08 (import direction), scaling UB by `abs(-0.08) × decay` constrains only the export direction. The import direction continues unconstrained. For mitochondrial substrate importers (PYRt2m, GLCt1r, etc.) under Scenarios B/C, this means substrates continue flowing in at full strength as proteins decay.

**Action:** Track signed baseline flux; scale both `upper_bound` and `lower_bound` directionally.

## AGENT CLAIMS THAT WERE OVERSTATED

### 6. "12h half-life is wrong — literature median is 96h" — PARTIALLY WRONG
Agent 3 claimed in vivo ETC half-lives cluster at 4-17 days (Kim 2012 cardiac: 17 days; Fornasiero 2018 brain: 9-20 days), invalidating the 12h assumption.

**Verified with literature search:** The in vivo claim is correct. BUT post-extraction conditions radically alter kinetics:
- Nuclear import of replacement subunits ceases entirely
- ROS production increases 5-20× without cytosolic glutathione/thioredoxin
- Lon/ClpXP proteases preferentially degrade oxidized ETC subunits
- Published functional decay timescales for isolated mitochondria: **2-24 hours**

**Verdict:** 12h is DEFENSIBLE for extracted mitochondria if we justify it explicitly with post-extraction context (citing functional decay literature, not in vivo turnover). It is INDEFENSIBLE if we imply it represents in situ protein turnover.

**Action:** Add a justification paragraph in abstract methods: *"Protein half-lives for extracted mitochondria were modeled at t½ = 12h, within the 2-24h functional decay range reported for isolated mitochondria (refs). In situ half-lives (4-17 days) are not applicable post-extraction due to loss of nuclear-encoded import machinery, elevated ROS, and cytosolic proteostasis dependency."*

### 7. Early exit in Exp 1c introduces non-monotonicity — TECHNICALLY TRUE, PRACTICALLY IRRELEVANT
Agent 1 flagged that the `if atp < threshold * 0.01: break` line could introduce artifacts in t½ ≤ 4h runs. Verified by inspection of `halflife_sweep.csv` — curves at t½ = 1h and 2h are cleanly monotonic; no observed artifact.

**Verdict:** Technically present; no measurable impact on published results.

## Addendum — Corrections found after writing the audit

### Correction 1: Gene KO stage has a DIFFERENT bug than the uniform-decay stage

The uniform-decay GPR bug (Issue #4) is latent — all 91 AND-linked genes compute the same decay factor, so last-write-wins gives the same answer. But the **gene knockout stage is different**:

```python
for rxn in gene.reactions:
    rxn.upper_bound = 0.0
```

- For **AND-rule reactions** (like CI_mitoMap, 91 subunits AND-linked): KOing any single gene correctly kills the reaction. ✓
- For **OR-rule reactions** (isozymes, substitutable subunits): KOing one OR-gene should NOT kill the reaction because the OR-partner still catalyzes. Our code sets UB=0 anyway → **over-kills OR reactions → inflates apparent essentiality → deflates the 348 dispensable count.**

**Consequence:** The "348/769 dispensable" number is a LOWER BOUND. True count under correct GPR is likely higher. Must re-run with `efflux_method.py` parsing before quoting any specific N in the abstract.

### Correction 2: PMF is NOT conserved in steady-state FBA

My proposed fix for Issue #2 ("maximize PMF_m accumulation") is wrong at the FBA level. In steady-state FBA, every metabolite's production rate equals its consumption rate at optimality — you cannot "accumulate" PMF because it's constrained to zero net flux.

**Correct formulation of Fix #2:**
- Block `ATPtmB_mitoMap` (set bounds to `(0, 0)`) — prevents PMF drain via ATP export to cytoplasm
- Set objective to maximize **combined proton-pumping flux** (CI + CIII + CIV) — the rate of PMF *generation*, which is the biophysically correct ΔΨm-maintenance metric under constraint-based modeling
- Alternatively: add an artificial "PMF leak" sink with a small fixed flux, and optimize for the rate of PMF generation that can sustain it

This changes the implementation but not the principle of Fix #2.

### Correction 3: Order of operations matters — no abstract reframing before bug fixes

Original audit recommended reframing the abstract around 348/769 + scaling law. This was premature. Both numbers come from buggy code:
- 348/769 is from KO stage with over-killed OR reactions → lower bound only
- Scaling law `window ≈ 2.4 × t½` derived under `OF_ATP_mitoMap` objective (wrong) with flux-relative scaling (our invented method). Under ΔΨm objective + E-Flux, the constant may change or the functional form may differ.

**Correct sequence: fix → re-derive → reframe → write.** Quoting any current number in an abstract on the grounds that "the audit supports it" would be compounding error.
